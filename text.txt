## **Step 3: Add GitHub remote**


## **Step 4: Attempt to push via HTTPS**

You ran:

```bash

```

❌ Mistake: Failed with SSL certificate error:

```

```

**Why:**

* Git on Windows couldn’t verify the SSL certificate.

**Correction:** Switch to **SSH** (more reliable on Windows).

## **Step 5: Generate SSH key**

You ran:

```bash
ssh-keygen -t ed25519 -C "maamiyee1210@gmail.com"
```

* When prompted, you overwrote old key (ok).
* You set an optional passphrase (secure) or left empty.

**Output:**


## **Step 6: Add SSH key to SSH agent**

```bash
eval $(ssh-agent -s)
ssh-add ~/.ssh/id_ed25519
```

**Output:**

```
Identity added: /c/Users/maami/.ssh/id_ed25519 (maamiyee1210@gmail.com)
```

---

## **Step 7: Copy public key to GitHub**

```bash
cat ~/.ssh/id_ed25519.pub
```

* Copy the output (starts with `ssh-ed25519 ...` and ends with your email).
* Go to GitHub → **Settings → SSH and GPG keys → New SSH key**
* Paste key, give it a title (e.g., `My PC`), click **Add SSH key**.

**Mistake before:** You didn’t add the key, so SSH push failed.

---

## **Step 8: Test SSH connection**

## **Step 9: Switch remote to SSH**

```bash

```

**Why:** This avoids HTTPS SSL issues.

## **Step 11: Handling remote conflicts**

When you tried:

```bash
git push -u origin main
```

❌ Error:

```
Updates were rejected because the remote contains work that you do not have locally.
```

**Why:**

* Remote repo had a README or other initial commits.

**Correction:** Pull and merge remote changes:

```bash
git pull origin main --allow-unrelated-histories
```

---

### **Step 12: Resolve merge conflicts**

* Git showed MERGING state.
* You checked:

```bash
git status
```

Output:

```
All conflicts fixed but you are still merging.
  (use "git commit" to conclude merge)
Changes to be committed:
        new file:   README.md
```

**Correction:** Commit the merge:

```bash
git commit -m "Merge remote changes from GitHub"
```

---

## **Step 13: Push final changes**

```bash
git push -u origin main
```

✅ Success:

```
branch 'main' set up to track 'origin/main'.
```

* Your project is now on GitHub.
* Future pushes are simple:
